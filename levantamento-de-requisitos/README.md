# 📋 Aula: Engenharia de Requisitos - Coleta e Classificação

## 1. O que são Requisitos?
Requisitos são as necessidades e condições que um sistema deve atender para satisfazer os objetivos do cliente.

---

## 2. Classificação de Requisitos

### 🛠️ Requisitos Funcionais (RF)
Descrevem **o que** o sistema deve fazer. São as funcionalidades e serviços.
*   *Exemplo:* "O sistema deve permitir que o usuário recupere sua senha por e-mail."
*   *Exemplo:* "O sistema deve gerar um relatório mensal de vendas em PDF."

### 🛡️ Requisitos Não Funcionais (RNF)
Descrevem **como** o sistema deve realizar suas funções. Estão ligados a qualidades, restrições e atributos do sistema.
*   **Desempenho:** "A página deve carregar em menos de 2 segundos."
*   **Segurança:** "As senhas devem ser armazenadas com criptografia SHA-256."
*   **Disponibilidade:** "O sistema deve estar online 99,9% do tempo."

---

## 3. Técnicas de Levantamento de Requisitos

### 🧠 Brainstorming
Técnica de "tempestade de ideias" para explorar o máximo de possibilidades sem julgamentos imediatos.
*   **Objetivo:** Gerar ideias inovadoras e descobrir requisitos ocultos.
*   **Dica:** Envolva stakeholders de diferentes áreas para ter visões variadas.

### 🎤 Entrevista
Conversa direta com os usuários e donos do negócio.
*   **Estruturada:** Perguntas fechadas e roteiro rígido.
*   **Não Estruturada:** Conversa aberta para explorar o domínio do problema.
*   **Dica:** Sempre pergunte "Por que?" para entender a real dor do usuário.

### 🎨 Prototipagem
Criação de modelos visuais do sistema antes do desenvolvimento real.
*   **Baixa Fidelidade:** Desenhos em papel (wireframes).
*   **Alta Fidelidade:** Protótipos interativos (Figma, Adobe XD).
*   **Vantagem:** Reduz erros de interpretação e valida a interface cedo.

## 📝 Exercício Prático
**Cenário:** Um aplicativo de entrega de comida.

1. Liste **2 Requisitos Funcionais**.
2. Liste **2 Requisitos Não Funcionais** (um de segurança e um de performance).
3. Qual técnica (Entrevista ou Brainstorm) você usaria para definir as formas de pagamento? Justifique.
